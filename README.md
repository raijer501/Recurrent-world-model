## Project Data: Understanding Rollouts

The training pipeline for this World Model relies on pre-collected gameplay sequences called **rollouts**. These are generated by running an exploration policy (initially selecting random actions) inside the Gymnasium `CarRacing-v3` environment.

Each rollout is stored as a compressed NumPy binary archive (`.npz`) inside the `data/rollouts/` directory to save space while preserving high-dimensional array structures.

### Data Dictionary & Shapes

When you load a rollout file (e.g., `rollout_0.npz`), it contains two primary tracking arrays:

1. **`frames`** (Shape: `(N, 96, 96, 3)`)
   * **`N`**: The total number of time-steps or video frames recorded during the episode (typically capped at 1,000 steps).
   * **`96, 96`**: The pixel resolution (Height x Width) of the car racing track observation.
   * **`3`**: The color channels (Red, Green, Blue) scaled from 0 to 255.
   * **Purpose**: Serves as the raw image dataset used for unsupervised training of the **Vision Module (VAE)**.

2. **`actions`** (Shape: `(N, 3)`)
   * **`N`**: The matching number of time-steps.
   * **`3`**: The continuous control inputs applied by the agent at that specific frame:
     * Index 0: Steering (-1.0 to 1.0)
     * Index 1: Gas (0.0 to 1.0)
     * Index 2: Brake (0.0 to 1.0)
   * **Purpose**: Passed alongside the visual states to train the **Memory Module (MDN-RNN)** so it can learn how specific driver actions alter the future physics of the world.



## Vision Module (V) Architecture

The **Vision Module** serves as the sensory component of our world model. It implements a **Variational Autoencoder (VAE)** tasked with compressing the high-dimensional visual track observations into a low-dimensional latent vector. This forces the model to strip away pixel noise and focus strictly on vital spatial structures like track curves and car positioning.

### Network Layout Summary

* **Input Format:** Pixel-normalized float tensor of shape `(Batch, 3, 96, 96)`

#### 1. Encoder (Compression Stage)
The encoder passes the image through four Convolutional layers, decreasing spatial resolution while increasing feature channels:
* **Layer 1:** `Conv2d(3 → 32)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(32, 48, 48)` + ReLU
* **Layer 2:** `Conv2d(32 → 64)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(64, 24, 24)` + ReLU
* **Layer 3:** `Conv2d(64 → 128)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(128, 12, 12)` + ReLU
* **Layer 4:** `Conv2d(128 → 256)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(256, 6, 6)` + ReLU
* **Flatten:** Flattens the final feature map into a 1D vector of size `9216`.

#### 2. Bottleneck (Latent Space)
The flattened features map into two parallel linear layers that parameterize the latent distribution:
* **Mean Layer (`fc_mu`):** `Linear(9216 → 32)`
* **Log Variance Layer (`fc_logvar`):** `Linear(9216 → 32)`
* **Reparameterization Trick:** Samples a latent vector $z$ (dimension: 32) using $z = \mu + \epsilon \cdot \sigma$ (where $\epsilon \sim \mathcal{N}(0, I)$), which allows gradient backpropagation through a random sampling layer.

#### 3. Decoder (Reconstruction Stage)
The decoder projects the latent sample $z$ back into a full-scale image to ensure the compressed vector accurately captures the environment's state:
* **Projection Layer:** `Linear(32 → 9216)` followed by an `Unflatten` operation back to `(256, 6, 6)`.
* **Layer 1:** `ConvTranspose2d(256 → 128)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(128, 12, 12)` + ReLU
* **Layer 2:** `ConvTranspose2d(128 → 64)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(64, 24, 24)` + ReLU
* **Layer 3:** `ConvTranspose2d(64 → 32)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(32, 48, 48)` + ReLU
* **Layer 4:** `ConvTranspose2d(32 → 3)` | Kernel: 4x4, Stride: 2, Padding: 1 → Output: `(3, 96, 96)` + Sigmoid Activation





## Memory Module (M) Architecture

The **Memory Module** serves as the predictive engine of the world model, responsible for capturing the temporal dynamics and physics of the environment over time. While the Vision Module compresses static snapshots of the track, the Memory module predicts what will happen next. It is implemented as a **Mixture Density Network combined with a Recurrent Neural Network (MDN-RNN)**.

Its core purpose is to predict the full probability distribution of the *next* latent state $z_{t+1}$ given the current latent state $z_t$, the current action $a_t$, and its own internal historical hidden state $h_t$:

$$P(z_{t+1} \mid a_t, z_t, h_t)$$

### Network Layout Summary

* **Input Format:** Two sequential tensors—$z_t$ shape `(Seq_Len, Batch, 32)` and $a_t$ shape `(Seq_Len, Batch, 3)`—concatenated along the feature dimension to form an input vector of size **35**.

#### 1. Temporal Processing (LSTM Layer)
The historical timeline of the episode is processed using a Long Short-Term Memory (LSTM) network:
* **LSTM Layer:** `LSTM(input_size=35, hidden_size=256)`
* **Sequence Handling:** It tracks a hidden state vector $h_t$ and a cell state vector $c_t$ across all 1,000 timesteps, encoding the car's momentum, velocity, and track history.

#### 2. Mixture Density Network (MDN) Output Heads
Because the car's future trajectory can be highly unpredictable (stochastic), the network does not output a single guess. Instead, the LSTM output maps to three parallel fully connected layers representing a mixture of **5 distinct Gaussian distributions**:
* **Mixing Coefficients (`fc_pi`):** `Linear(256 → 5)` + Softmax activation. Outputs the probability ($\pi$) or "weight" of each of the 5 Gaussian components.
* **Gaussian Means (`fc_mu`):** `Linear(256 → 5 * 32)` reshaped to `(5, 32)`. Outputs the expected coordinates ($\mu$) for the next latent vector across all 5 possibilities.
* **Gaussian Variances (`fc_sigma`):** `Linear(256 → 5 * 32)` reshaped to `(5, 32)` + Exponential activation (`torch.exp`). Outputs the strictly positive standard deviations ($\sigma$), tracking the model's spatial uncertainty.

### Operational Output Shapes
The output shapes preserved across the entire time sequence look like:
* **`pi` (Weights):** `(Seq_Len, Batch, 5)`
* **`mu` (Means):** `(Seq_Len, Batch, 5, 32)`
* **`sigma` (Stds):** `(Seq_Len, Batch, 5, 32)`



## Controller Module (C) Architecture

The **Controller Module** is the decision-making driver of the world model. It is intentionally designed as a minimal, single-layer linear network. Because the Vision module handles spatial complexity and the Memory module handles temporal physics, the Controller does not need to be deep. It simply maps the concatenated outputs of the VAE ($z_t$) and RNN ($h_t$) directly to physical driving controls.

### Network Layout Summary

* **Input Format:** Two flat state tensors—$z_t$ shape `(Batch, 32)` and $h_t$ shape `(Batch, 256)`—concatenated to create a combined state vector of size **288**.

#### 1. Linear Mapping Layer
* **Layer:** `Linear(288 → 3)`
* **Purpose:** Correlates the full perception-memory footprint directly to steering, accelerating, and braking signals.

#### 2. Action Activation Slicing
To ensure the raw floating-point calculations match the exact operational bounds required by Gymnasium's `CarRacing-v3` physical constraints, the outputs are separated and passed through specific activation functions:
* **Steering Component:** Passed through `torch.tanh` to bound values strictly between **-1.0** (full left lock) and **1.0** (full right lock).
* **Gas Component:** Passed through `torch.sigmoid` to bound values strictly between **0.0** (no gas) and **1.0** (full acceleration).
* **Brake Component:** Passed through `torch.sigmoid` to bound values strictly between **0.0** (no brake) and **1.0** (maximum stopping power).

### Operational Output Shapes
* **`action`:** `(Batch, 3)` containing `[steering, gas, brake]`.


## 4. Controller Module (C) Mathematical & Boundedness Framework

The **Controller Module** is the decision-making driver. It is designed as a minimal, single-layer linear network that maps the concatenated outputs of the VAE ($z_t$) and RNN ($h_t$) directly to physical driving controls.

### Core Mathematical Equations
The controller executes an affine transformation followed by an element-wise activation vector mapping at each discrete time-step $t$:

$$I_t = \begin{bmatrix} z_t \\ h_t \end{bmatrix}$$

$$y_t = W_c I_t + b_c$$

$$a_t = \Phi(y_t) = \begin{bmatrix} \tanh(y_{t,1}) \\ \sigma(y_{t,2}) \\ \sigma(y_{t,3}) \end{bmatrix}$$

### Variables and Parameters Glossary

#### Dimensions (Positive Constants)
*   $N_z$: Dimension of the latent space representing visual perception ($N_z = 32$).
*   $N_h$: Dimension of the RNN hidden state representing memory ($N_h = 256$).
*   $N_a$: Dimension of the control action space ($N_a = 3$).

#### Time-Varying Variables
*   $t$: Current discrete time-step of the simulation.
*   $z_t \in \mathbb{R}^{N_z}$: Latent vector generated by the VAE at time $t$, capturing spatial features.
*   $h_t \in \mathbb{R}^{N_h}$: Hidden state vector generated by the LSTM at time $t$, capturing temporal dynamics.
*   $I_t \in \mathbb{R}^{N_z + N_h}$: Concatenated state input vector at time $t$ (dimension: $32 + 256 = 288$).
*   $y_t \in \mathbb{R}^{N_a}$: Raw logit vector (pre-activation outputs) before bounding operations are applied.
*   $y_{t,1}, y_{t,2}, y_{t,3}$: Individual scalar components of the raw logit vector $y_t$.
*   $a_t \in \mathbb{R}^{N_a}$: Final bounded action vector executed in the environment, where $a_t = [a_{t,1}, a_{t,2}, a_{t,3}]^T$.
    *   $a_{t,1}$: Steering command.
    *   $a_{t,2}$: Throttle (gas) command.
    *   $a_{t,3}$: Braking command.

#### Trainable Parameters (Static Coefficients)
*   $W_c \in \mathbb{R}^{N_a \times (N_z + N_h)}$: Weight matrix of the linear layer (dimension: $3 \times 288$).
*   $b_c \in \mathbb{R}^{N_a}$: Bias vector of the linear layer (dimension: $3 \times 1$).

#### Non-Linear Operators
*   $\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$: Hyperbolic tangent function used to scale steering.
*   $\sigma(x) = \frac{1}{1 + e^{-x}}$: Standard logistic sigmoid function used to scale gas and braking.

### Mathematical Proof of Boundedness

To guarantee that the controller's computed action $a_t$ is bounded and cannot produce out-of-bounds driving inputs, we prove that the function maps any arbitrary input domain to a compact set:

#### Step 1: Boundedness of Inputs ($I_t$)
1.  **Memory Vector ($h_t$):** Computed via the LSTM output gate formulation $h_t = o_t \odot \tanh(c_t)$. Because the range of the hyperbolic tangent function is explicitly bounded, $\tanh(x) \in (-1, 1)$, every element of the hidden state vector is strictly bounded: $\|h_t\|_\infty < 1$.
2.  **Vision Vector ($z_t$):** The raw input space consists of pixel arrays restricted to the compact domain $[0, 255]^d$. Because the VAE encoder is a continuous mapping function composed entirely of convolutions and ReLU activations, it maps a compact set to a compact set. Consequently, there exists a finite real constant $M_z$ such that: $\|z_t\|_\infty \leq M_z < \infty$.

Since both sub-components are bounded, the total concatenated input vector satisfies $\|I_t\|_\infty < \infty$.

#### Step 2: Finiteness of the Linear Map ($y_t$)
Assuming regularized parameter optimization (such as weight decay or bounded evolutionary updates), the parameter sets remain finite such that $\|W_c\|_\infty < \infty$ and $\|b_c\|_\infty < \infty$. The linear combination of bounded terms yields a finite logit vector:

$$\|y_t\|_\infty = \|W_c I_t + b_c\|_\infty < \infty$$

#### Step 3: Global Boundary Saturation via Asymptotic Limiting
Even if unexpected numerical scale shifts cause the intermediate pre-activation outputs $y_t$ to diverge ($y_{t,i} \to \pm\infty$), the element-wise boundary mapping operator $\Phi(y_t)$ guarantees strict global truncation due to its definitive horizontal asymptotes:

$$\lim_{x \to \infty} \tanh(x) = 1, \quad \lim_{x \to -\infty} \tanh(x) = -1 \implies \tanh(y_{t,1}) \in [-1, 1]$$
$$\lim_{x \to \infty} \sigma(x) = 1, \quad \lim_{x \to -\infty} \sigma(x) = 0 \implies \sigma(y_{t,i}) \in [0, 1] \quad \text{for } i \in \{2, 3\}$$
4
#### Verification Conclusion
The controller output $a_t$ is mathematically guaranteed to reside within the closed, compact hypercube metric space $\mathcal{A}$:

$$a_t \in [-1, 1] \times [0, 1] \times [0, 1] \subset \mathbb{R}^3$$

This system bound remains invariant regardless of any transient trajectory changes or instabilities occurring within the upstream recurrent memory states.


## 5. Vision Module (VAE) Training Details

The Vision Module ($V$) is optimized using an unsupervised training routine that processes thousands of independent track frames extracted from saved environment rollouts.

### Data Ingestion pipeline
* **Tensor Formatting:** Raw rollout frames are stored in Height-Width-Channel (`HWC`) format. The dataset class permutes these into Channel-Height-Width (`CHW`) format using `.permute(2, 0, 1)` to align with PyTorch convolutional layer requirements.
* **Normalization:** Image pixel integers within $[0, 255]$ are cast to float32 and normalized down to the compact range $[0.0, 1.0]$ via a $255.0$ division step to ensure stable weight gradients.

### The Objective Loss Function
The model optimizes weights using a composite loss function tracking two opposing mathematical forces:

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{recon}} + \mathcal{L}_{\text{KLD}}$$

1. **Reconstruction Loss ($\mathcal{L}_{\text{recon}}$):** Measures the absolute pixel-level Mean Squared Error (MSE) between the target frame $x$ and the decoded reconstruction $\hat{x}$. It enforces structural clarity (tracking lines, curves, and car orientation):
   $$\mathcal{L}_{\text{recon}} = \sum_{i=1}^{D} (x_i - \hat{x}_i)^2$$

2. **Kullback-Leibler Divergence ($\mathcal{L}_{\text{KLD}}$):** Acts as a spatial regularizer, calculating the statistical divergence between the encoder's generated latent distribution $\mathcal{Q}(z \mid x) = \mathcal{N}(\mu, \sigma^2)$ and the prior standard normal distribution $\mathcal{P}(z) = \mathcal{N}(0, I)$:
   $$\mathcal{L}_{\text{KLD}} = -\frac{1}{2} \sum_{j=1}^{N_z} \left(1 + \log(\sigma_j^2) - \mu_j^2 - \sigma_j^2\right)$$

### Optimization Configuration
* **Optimizer:** Adam ($\alpha = 0.001$)
* **Batch Size:** 64
* **Target Boundaries:** The final decoder layer leverages a Sigmoid function, clamping output values strictly within $[0.0, 1.0]$ to perfectly match the target domain.
* **Output Artifact:** Weights are serialized at each epoch checkpoint to `checkpoints/vae.pth`.
